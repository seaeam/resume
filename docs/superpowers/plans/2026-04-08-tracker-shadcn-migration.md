# Tracker Shadcn Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the tracker page's hand-written switcher UI with direct shadcn `ToggleGroup` and `Tabs` usage, without changing tracker business logic or adding a new wrapper component layer.

**Architecture:** Keep the existing Zustand store, drawer open/close flow, and tracker business components intact. Swap only the interaction carrier layer: `ViewToggle` and `StatusFilter` become controlled `ToggleGroup`s, and `JobDrawer` owns shadcn `Tabs` directly while the custom `DrawerNav` file is removed.

**Tech Stack:** React 19, TypeScript, Vite, Zustand, shadcn/ui (radix), Tailwind CSS v4, pnpm, ESLint

---

## Task 2: Replace the Status Filter with `ToggleGroup`

**Files:**
- Modify: `src/pages/tracker/components/status-filter/index.tsx`
- Reference: `src/components/ui/toggle-group.tsx`
- Reference: `src/components/ui/badge.tsx`
- Reference: `src/pages/tracker/const.ts`
- Reference: `src/pages/tracker/types.ts`

- [ ] **Step 1: Normalize the `null` filter state into a local `all` sentinel for the control value**

Keep store state unchanged, but map it at the component edge:

```tsx
const activeValue = filterStatus ?? 'all'
```

and reverse it in `onValueChange`:

```tsx
onValueChange={(value) => {
  if (!value)
    return
  setFilterStatus(value === 'all' ? null : (value as ApplicationStatus))
}}
```

- [ ] **Step 2: Replace the hand-written button list with shadcn `ToggleGroupItem`s**

Render the filter bar as a single-select toggle group. Keep layout-only classes on the container, and use `Badge` for counts instead of a custom count pill:

```tsx
<ToggleGroup
  type="single"
  value={activeValue}
  variant="outline"
  className="scrollbar-gutter-stable w-full justify-start overflow-x-auto"
  onValueChange={...}
>
  {ALL_FILTER_STATUSES.map((status) => {
    const value = status ?? 'all'
    const label = status === null ? '全部' : APPLICATION_STATUS_CONFIG[status].label
    const count = getCount(status)

    return (
      <ToggleGroupItem key={value} value={value}>
        <span>{label}</span>
        <Badge variant="secondary">{count}</Badge>
      </ToggleGroupItem>
    )
  })}
</ToggleGroup>
```

- [ ] **Step 3: Keep horizontal scrolling behavior without recreating a custom control style**

Preserve only the existing container behavior that matters for overflow:

- horizontal scroll on narrow screens
- hidden scrollbar styling already used in tracker
- no hand-written active/inactive button color system
