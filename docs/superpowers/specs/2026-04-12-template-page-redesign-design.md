# Template 页面产品化重构设计

**日期：** 2026-04-12

**目标：** 将 `/template` 从“功能可用但页面感不足”的模板入口，重构为一个更完整的模板中心与编辑工作台，在不改变现有模板数据流和核心交互语义的前提下，提升样式、布局、图标使用和信息层级，让页面看起来像真正的产品页面。

---

## 问题定义

当前模板页的问题不在于功能缺失，而在于页面表达不够成熟：

- 模板库页目前更像“标题 + 单个容器卡片”，缺少页面级头部、内容层级和产品感。
- 官方模板、社区模板、我的模板虽然已经分 tab，但信息分层较弱，用户进入后很难快速建立“这里能做什么”的心智。
- 编辑器已经具备左中右三栏能力，但整体仍偏功能拼接，工具栏、结构面板、画布、属性面板之间缺少统一的工作台气质。
- 加载态、空状态、错误态和状态徽标虽然有基础支持，但还没有形成完整、统一的页面语言。
- 模板页作为产品主入口之一，视觉完成度目前低于 dashboard、tracker 等已有页面。

这次重构的目标不是新增模板功能，而是把已有能力包装成更成熟、更可信的产品体验。

---

## 用户确认的方向

本次设计基于以下已确认约束：

- 同时优化 **模板库页** 和 **模板编辑器**。
- 视觉方向为 **现代产品化、精致但克制**。
- 允许做 **页面级重组**，但不改底层数据逻辑。
- 页面与组件实现必须 **使用现有 shadcn 体系**，不额外发明一套自定义基础组件。

---

## 设计原则

### 1. 页面先像产品，再像功能集合

用户进入 `/template` 后，应该先感受到“这是模板中心 / 工作台”，再开始点选具体模板或进入编辑。页面级头部、分区、摘要、状态反馈必须先建立完整产品心智。

### 2. 保留现有工作流，升级表达方式

本次不改变模板加载、保存、发布、复制、预览的数据流和业务语义。改造重点是：

- 页面层级
- 卡片结构
- 操作层次
- 图标语言
- 状态反馈
- 编辑器 workspace 感

### 3. 模板库与编辑器共享同一种视觉语言

模板库页偏“浏览与选择”，编辑器偏“操作与配置”，但两者应该被识别为同一产品子系统：

- 一致的标题层级
- 一致的状态表达
- 一致的卡片和分区节奏
- 一致的图标与按钮语义

### 4. 现代感来自层级与秩序，不来自重装饰

页面应避免过度炫技的渐变、强烈纹理或大面积装饰块。视觉升级主要通过以下方式完成：

- 更清晰的主次层级
- 更稳定的留白节奏
- 更完整的 header / summary / section 结构
- 更可信的状态、提示和操作编排

### 5. 只用现有 shadcn 组件体系做组合

本次实现必须限定在项目当前的 shadcn 配置内：

- `style: new-york`
- `base: radix`
- `iconLibrary: lucide`

优先复用现有组件：

- `Card`
- `Tabs`
- `Badge`
- `Button`
- `Empty`
- `Alert`
- `Skeleton`
- `Separator`
- `ScrollArea`
- `Tooltip`
- `ToggleGroup`
- `Spinner`

不新增自定义基础容器，不用 raw Tailwind 重新造通用 UI。

---

## 页面级架构

`/template` 继续保留两种模式，但表达方式升级：

### 模式 1：模板库页

模板库页应被重构成真正的“模板中心”，包含：

1. 页面头部（标题、说明、主操作、摘要）
2. 模板分发区（官方 / 社区 / 我的）
3. 模板卡片网格
4. 完整的空状态、加载态、错误态

### 模式 2：模板编辑器

模板编辑器保留现有三栏工作台结构，但重新组织成统一 workspace：

1. 顶部主控台
2. 左侧结构面板
3. 中间实时画布
4. 右侧属性面板

这样的架构既保留当前实现基础，也能显著提升页面完成度。

---

## 模板库页重构

## 1. Hero 头部

模板库页顶部不再只是标题与一句文案，而是一个更完整的 shadcn 头部卡片。

建议结构：

- 左侧：
  - 页面标题：`简历模板中心`
  - 说明文案：突出“直接使用 / 自定义 / 沉淀我的模板”
  - 辅助提示：告诉用户这是模板选择与模板沉淀入口
- 右侧：
  - 主操作：例如“从官方模板开始”
  - 次操作：例如“继续编辑我的模板”
- 底部摘要：
  - 官方模板数量
  - 社区可用模板数量
  - 我的模板数量

摘要区不需要做成统计看板，而是用轻量卡片或 badge 组合，提升页面的系统感和可读性。

“继续编辑我的模板”需要有明确回退逻辑：

- 如果用户已有最近编辑的个人模板，则直接进入该模板的编辑器
- 如果用户有多个模板但没有明确最近项，则落到“我的模板”tab
- 如果用户还没有个人模板，则替换为“创建我的第一个模板”或跳转到官方模板入口

## 2. Tab 导航提升为频道切换

当前 tabs 可保留，但需要升级成更像频道导航的表达：

- 每个 tab 增加图标
- 每个 tab 有更清晰的定位文案
- 选中态更明显
- 与上方头部形成明确的内容承接

目标不是改变 tab 逻辑，而是让三类模板的差异更容易被理解。

## 3. 模板区块增强

每个 tab 内都应具备独立的信息层次：

- 小标题或辅助描述
- 模板网格
- 必要时的轻量提示
- 独立空状态

模板卡片保持当前能力，但强化以下几点：

- 缩略图更像真实模板封面，而不是单纯预览切片
- 标签组合更稳定，清楚表达布局、风格、来源、发布状态
- 主动作更明确：直接使用 / 自定义 / 继续编辑 / 发布
- 删除、取消发布等低频动作保持次级层级

## 4. 页面状态完整化

模板库页所有反馈都应页面化：

- 加载：使用 `Skeleton` 作为模板卡片骨架
- 空状态：统一使用 `Empty`
- 错误：使用 `Alert` 或带 header 的 `Card`

不要再出现“页面中间一行字”的弱反馈形式。

## 模板编辑器重构

## 1. 顶部工具栏升级为主控台

当前工具栏已经具备完整功能，但视觉上仍偏工具条。改造后应变成编辑器的主控台：

- 左侧展示：
  - 模板名称
  - 当前状态（未保存 / 私有 / 已发布）
  - 一句轻量说明
- 右侧展示：
  - 返回列表
  - 恢复默认
  - 另存为副本
  - 保存

状态表达优先使用 `Badge`，按钮遵循现有 shadcn variant 体系，图标统一使用 lucide 并按规则加 `data-icon`。

现有的私有 / 发布控制必须继续保留在顶栏主控台中，不能被折叠进低优先级菜单或次级面板；因为它属于模板状态的一部分，而不是低频管理动作。

## 2. 左侧结构面板像导航面板，而不是普通列表

结构面板应进一步强化“这是模板结构导航”的感觉：

- 主栏 / 侧栏 / 模块库拥有更清晰的标题、数量、说明
- 每个区块之间用 `Separator` 拉开节奏
- section 项维持现有拖拽能力，但视觉上更稳定、更易扫描
- 当前选中 section 的高亮和上下文感更明确

左侧的目标不是更华丽，而是更好用、更像真正的结构树入口。

## 3. 中间画布明确为主任务区

中间预览画布是编辑器的核心，应获得最强视觉聚焦：

- 预览简历切换器成为画布控制条的一部分
- 画布卡片边界、背景衬底和滚动区层级更清楚
- loading 状态不再只是文案，而是有骨架或稳定占位

这能让用户在进入编辑器时立即知道“中间是成果区，两侧是控制区”。

## 4. 右侧属性面板做成稳定的双模式

属性面板存在两种上下文：

- 模板属性
- 模块属性

改造后应让这两种上下文的切换更明显，例如通过标题、副标题、返回模板属性动作来强调当前模式。
目标是减少“我现在是在改整体还是改单个模块”的认知成本。

---

## 图标与操作语言

本次会统一模板页的图标语义，但保持克制：

- 页面级头部使用少量识别图标，帮助区分模板中心、模板库、编辑器工作台
- tab、主操作、状态徽标使用一致图标语言
- 按钮内图标遵循 shadcn 规则，不手工加尺寸类
- 不用图标堆砌视觉噪音，只在帮助识别层级和动作时使用

交互语义保持不变，但层级重新整理：

- 主要动作一眼可见
- 次级动作降级
- 破坏性动作继续放在确认流程中
- 私有 / 发布 / 未保存等状态用稳定徽标表达

---

## 数据流与实现边界

本次重构不改动以下链路：

- `useTemplateWorkbenchStore`
- `useTemplateEditorStore`
- 模板加载、复制、保存、发布逻辑
- 模板实时预览链路
- `ResumeTemplateRuntime` 渲染机制

实现上只做页面和组件组织层面的升级，不做：

- 模板 schema 修改
- Supabase 表结构调整
- 模板行为语义变更
- 新的模板业务流程

---

## 状态与反馈规范

页面内所有关键状态必须统一表达：

- 加载态：`Skeleton`
- 空状态：`Empty`
- 错误态：`Alert` / `Card`
- 状态标识：`Badge`
- 保存中：`Spinner` + `Button`

目标是把模板页从“局部组件反馈”升级为“产品级反馈”。

---

## 非目标

本次不包含以下范围：

- 新增模板搜索、筛选、排序系统
- 改造模板底层数据模型
- 新增模板市场推荐算法
- 调整模板保存/发布业务规则
- 重写模板卡片缩略图渲染逻辑
- 引入新的设计系统或脱离 shadcn 的自定义基础组件库

---

## 验收标准

当以下条件同时成立时，本次设计目标算达成：

- 模板库页看起来像完整产品页，而不是一个包着 tabs 的大卡片
- 编辑器看起来像统一工作台，而不是三个独立模块横向摆放
- 官方 / 社区 / 我的模板三类内容更容易理解和切换
- 主要动作、次级动作、状态反馈层级更清晰
- 所有实现保持在现有 shadcn 体系内
- 现有模板创建、编辑、保存、发布、预览逻辑不受影响
